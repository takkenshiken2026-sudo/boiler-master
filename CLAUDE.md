# boiler-master 作業ルール（二級ボイラー技士マスター）

このリポジトリは二級ボイラー技士試験の学習サイト（`boiler-master.jp`）。ガイド記事は
`data/guide_articles.csv` を正本に `tools/build_article_pages.py` / `tools/build_all.py` で生成する。

## 0. 絶対ルール
1. **定型文・テンプレ禁止（GSCで弾かれる）。** 記事ごとに構成・切り口・見出し・文章・表の有無/数/順序を変える。
   同一フレーズ（例「10月11日13:30本番まで残り18週」等の日付詰め込み）を複数記事で使い回さない。
   FAQ・本文を字数合わせで機械的に埋めない（埋め草禁止）。
2. **一次情報で正確に。** 数値・制度は公式を起点に確認。記事内では「最新は公式で確認」を促す。

## 2. ビルダーの最重要クセ
- **section本文の1文に読点（、）は1つまで。** 2つ以上で自動的に `<ul>` 箇条書き化され文意が壊れる
  （`tools/seo_body_markup.py` の `inject_comma_sentence_list`：`sent.count("、") >= 2` で分解）。
  列挙は中黒「・」か表（markdown `|`）で。**lead / FAQ / user_intent は対象外**。
- 品質ゲート（published）：section本文 **≥180字** / FAQ回答 **≥100字**（自然に満たす。埋め草禁止）。
  字数判定は `tools/guide_article_rules.py` の `reader_facing_text` 基準。

## 3. 1記事の手順
1. テーマを調査し他記事と被らない切り口を決める。
2. `tools/rewrites/<slug>.py` に `REWRITES={slug:{...}}` で執筆。末尾に未使用セクション空化。
3. `python3 tools/apply_guide_rewrite_batch.py --batch tools/rewrites/<slug>.py`
4. `python3 tools/build_article_pages.py` → 検証 → 3〜4本ずつコミット。
5. 区切りで `python3 tools/build_all.py`（exit 0 必須）で全再生成し、再生成ページもコミット。
6. affiliate記事（affiliate-*）は商品・価格・ASPリンクを保持（構成の非テンプレ化のみ）。
